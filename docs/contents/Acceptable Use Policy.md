# HelloRun Acceptable Use Policy

**Last Updated:** May 23, 2026
**Platform:** HelloRun
**Website:** https://hellorun.online
**Contact:** hellorunonline@gmail.com, 4hprojects@proton.me

## 1. Purpose

This Acceptable Use Policy explains what users, organisers, and visitors may not do when using HelloRun.

It protects the platform, users, organisers, participant data, event records, and system security.

## 2. Prohibited Technical Activity

Users must not:

- attempt unauthorised access
- bypass login or access controls
- scrape data without permission
- overload the platform
- test vulnerabilities without written permission
- upload malware
- interfere with platform operation
- reverse engineer restricted platform systems
- bypass payment or verification workflows
- abuse APIs or automated requests
- use bots to create fake registrations or submissions

## 3. Prohibited Data Misuse

Users and organisers must not:

- collect participant data without permission
- export data for unrelated purposes
- sell or share platform data
- publish private payment proof
- expose another user’s personal information
- reuse uploaded proof outside its intended event purpose
- attempt to identify private users through scraped records

## 4. Prohibited Fraud

Users must not:

- submit fake run proof
- submit edited screenshots
- submit another person’s activity as their own
- reuse transaction proof dishonestly
- create multiple accounts to manipulate results
- impersonate organisers or participants
- claim awards, certificates, or rankings through false information

## 5. Prohibited Content and Behaviour

Users must not use HelloRun to post, upload, transmit, or promote:

- illegal content
- threats
- harassment
- hate speech
- spam
- scams
- malware
- phishing links
- sexually explicit content
- unauthorised personal data
- misleading event information
- content that infringes intellectual property rights

## 6. Security Testing

Users must not conduct vulnerability testing, penetration testing, scraping, load testing, or automated scanning against HelloRun systems without prior written permission.

Responsible security reports may be sent to HelloRun through the contact channels below.

## 7. Enforcement

HelloRun may take action when this policy is violated, including:

- warning the user
- rejecting submissions
- removing content
- hiding results
- removing leaderboard entries
- suspending features
- suspending accounts
- banning accounts
- restricting organiser access
- reporting illegal activity where appropriate
